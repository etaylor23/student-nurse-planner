## Goal

The first AI layer in PlaceMate: a student asks in plain English, their **own logged
note surfaces verbatim** (linked to the real entity), with a short grounded answer and
a gentle model-knowledge accuracy check. Runs on **AWS Bedrock** with a near-frontier
Claude model so **all charges land on the $1,000 AWS credits** — no non-AWS billing
anywhere in the path.

## Hard constraints

- **Bedrock only, credits only.** No third-party APIs (search, vector DBs, etc.) that
  bill outside AWS. This is why v1 has no live retrieval of external sources — there is
  no AWS-native web-search API, and Tavily/Brave/Google would bill separately.
- **Frontier / near-frontier model.**

## Decisions (locked in grilling, 2026-07-24)

| # | Decision |
|---|----------|
| D1 | **V1 scope: notes recall + model-knowledge accuracy check.** The model may use its clinical knowledge to sanity-check a note and to answer, but **never names/cites sources**. No external-source retrieval in v1. |
| D2 | **Grounding: model knowledge only.** Sonnet-class knowledge of NMC standards / five rights / ANTT etc. is sufficient at student level. **V2 options (spec note, not built):** Bedrock Knowledge Base with the built-in Web Crawler over an allowlist of NMC/NICE/RCN public pages (fully AWS-billed), curated corpus in S3, or fine-tuning — revisit when accuracy demands it. |
| D3 | **Retrieval: context-stuffing, no vector store.** A user's whole note corpus (est. 5–50k tokens) is fetched from DynamoDB and placed in the prompt. Verbatim fidelity is trivially guaranteed (the note is literally in context); the model returns the entity ID so the UI renders/links the real note. Revisit only if a corpus nears ~150k tokens. |
| D4 | **Corpus scope: everything except self-care.** Shift notes, reflections + Gibbs sections (**including locked reflections**), med logs, medication personal notes, skill/proficiency progress notes, tags. **Hard-excluded:** `SelfCareCheckin` entirely (note, energy, items — documented on-device/private promise, no recall value). |
| D5 | **Locked reflections: quote freely.** The lock remains a browsing-UI gate; a direct question is treated as intent to see the answer. No blur/unlock step in AI answers. |
| D6 | **Model: Claude Sonnet 5 on Bedrock** (`anthropic.claude-sonnet-5` via the Mantle client, or the EU cross-region inference profile on the legacy path — verify what eu-west-2 serves at build time; keep inference in-EU). **Prompt caching is required**: stable system prompt → per-user corpus with a `cache_control` breakpoint → volatile question last. Est. $0.01–0.05/question cached. |
| D7 | **Delivery: streaming from day one** via a Lambda **Function URL with response streaming** (SSE) — the existing HTTP API (API GW v2) cannot stream. **Condition:** auth must extend cleanly — verify the same Cognito access token in-Lambda (`aws-jwt-verify` against the existing user pool) + the same Cedar/AVP self-owned check. If that turns up a real blocker, fall back to non-streaming `ai/ask` on the existing router (29s ceiling). |
| D8 | **UX: the teaser becomes real** in its Home hero slot (badge flips "Coming soon" → "New"), and the **global ask bar** in the app shell opens the *same* component as an overlay — identical functionality everywhere. |
| D9 | **No-note / general-question policy: notes-first, confidence-gated.** Relevant note → verbatim quote + accuracy check. No note → clearly-labelled general answer **only when the model is confident**; guarded/hedged when not; plus a capture nudge ("log it on your next placement and I'll find it"). Always framed as *educational, check local policy* — never clinical instruction. |
| D10 | **"Find more" links: search-URL pattern only.** The model emits a topic phrase; the app builds a search URL on the authority's own site (NICE CKS `/search?q=…`, BNF search, NMC site search). The model never invents deep links — a search URL cannot 404 or misattribute. (Curated topic→URL map is a possible v1.1 upgrade.) |
| D11 | **Cost guardrails — full bundle:** per-user daily cap ~30 questions (DynamoDB counter item + TTL, friendly "back tomorrow" copy) · global kill switch (SSM/env flag checked per-request, flip without redeploy) · AWS Budgets alerts on Bedrock spend at $50/$150/$400 · `max_tokens` ≈ 1024. |
| D12 | **Q&A storage: full logging incl. operator access.** Persist question, answer, matched note refs, feedback, token usage per user in the single table. Operator may review for quality during beta. |
| D13 | **Disclosure: first-use notice + persistent caption.** Inline notice on first use — *"During beta, questions and answers are stored and may be reviewed by the PlaceMate team to improve accuracy"* — which auto-dismisses once they start using it (no acknowledge button), plus a one-line caption under the ask bar. |
| D14 | **Rollout: all signed-in users at deploy** (no per-user flag; kill switch + caps cover emergencies). **Guests keep the teaser** with copy along the lines of "sign in to use the full version". **Send the promised one-off launch email** to `aiRecallInterestAt` users via the existing `emails/` + `scripts/` beta tooling. |

## Architecture

```
SPA (shared AskNotes component: Home hero + global ask bar overlay)
  │  POST (SSE)  Authorization: <same Cognito access token>
  ▼
Lambda Function URL (response streaming, Node)
  1. aws-jwt-verify → same user pool          (auth unchanged, new enforcement point)
  2. Cedar/AVP self-owned action check + audit log (existing pattern)
  3. Kill-switch + daily-cap check (DynamoDB counter, TTL)
  4. Fetch corpus from DynamoDB (existing owner-partition queries; EXCLUDES self-care)
  5. Bedrock Sonnet 5 — system prompt │ corpus [cache_control] │ question
  6. Stream tokens back; persist AiQuestion record; emit usage metrics
```

- **Region:** eu-west-2 stack; Bedrock via EU inference (data stays in-EU; Bedrock does
  not train on inputs). One-time console step: enable Anthropic model access.
- **CORS:** Function URL locked to `https://app.placemate.uk` (+ localhost dev).
- **IAM:** the AI Lambda gets `bedrock:InvokeModelWithResponseStream` scoped to the
  Sonnet model/inference-profile ARNs, plus read on the table's owner partition.
- **Prompt/output contract:** model answers as structured segments — matched note
  (entity type + ID + verbatim quote), commentary/accuracy check, optional general
  answer (labelled), optional `find_more` topic phrase → app renders note card linked
  to the real entity + search-URL chip.

## New data (single table, existing codegen flow)

- `AiQuestion` — userId, question, answer, matchedNoteRefs, feedback (`UP`/`DOWN`/null),
  inputTokens/outputTokens/cacheReadTokens, latencyMs, createdAt. User-owned partition.
- `AiDailyCount` — userId + date counter item with TTL (cap enforcement).
- `User.aiFirstUsedAt?` — drives the auto-dismissing first-use notice.
- (Existing `aiRecallInterestAt` powers the launch-email recipient list.)

## Implementation order

1. **Infra** — AI Lambda (streaming Function URL), IAM, SSM kill switch, Budgets
   alerts, CDK wiring. *Watch: `infra/lambda/**` typecheck gap — verify by hand.*
2. **Backend** — JWT verify + Cedar check, cap/kill-switch, corpus assembly
   (self-care excluded), Bedrock call with caching, streaming protocol, `AiQuestion`
   persistence, metrics/alarm on error rate.
3. **Domain/types** — new entities + `gen:zod`.
4. **Frontend** — convert `AiRecallTeaser` into live `AskNotes` (streaming render,
   note-card linking, search-URL chips, feedback thumbs, first-use notice + caption,
   cap/kill-switch/guest states); global ask bar in the shell opening the same
   component; guest teaser copy change.
5. **Prompting + eval pass** — system prompt (verbatim-quote contract, confidence
   gating, no-source-citing rule, tone per ethos: encouraging, "your notes not
   guidance"); manual eval against seeded corpora incl. adversarial dosing questions.
6. **Launch** — deploy, smoke-test live, send notify-me email, update runbook.

## Risks & mitigations

- **Hallucinated "verbatim" quotes** — the note is in-context and the contract is
  quote-by-ID; the app renders the note **from the DB by returned ID**, not from model
  output, so the quote shown is provably verbatim. Model text is commentary only.
- **Clinical-advice liability** — confidence-gated general answers, educational
  framing + check-local-policy line, no source citing (D1), search-URL links only
  (D10), eval pass includes dosing/med questions.
- **Cost runaway** — D11 bundle; caching asserted in tests via `usage.cache_read_input_tokens`.
- **Streaming auth blocker** — explicit fallback locked (D7): non-streaming on the
  existing router.
- **Privacy step-change (operator-readable Q&A)** — D13 disclosure; revisit before
  any wider (post-beta) rollout, likely adding an opt-out.
- **Model availability in eu-west-2** — verify Sonnet 5 serving path (direct vs EU
  cross-region profile) before building; if only a profile is offered, use it (in-EU).

## Acceptance criteria

- Signed-in student types a question anywhere (Home or global ask bar) → streamed
  answer; matched note rendered from the DB via returned entity ID, linked to source.
- No-note question → confident labelled general answer or guarded response + capture
  nudge; never a fabricated note; med-dosing probes answer educationally with the
  check-local-policy framing.
- Self-care data never leaves the device path (assert corpus assembly excludes it).
- Prompt cache hit on second question (nonzero `cache_read_input_tokens`).
- Daily cap and kill switch both verified live; Budgets alerts configured.
- First-use notice appears once and auto-dismisses; caption persists.
- Guests see the teaser with sign-in copy; notify-me email sent to interest list.
- All Bedrock charges visible under the AWS account (credits) — nothing billed elsewhere.
