Colour palette guide

A calm, professional system for a student-nurse companion app. Emerald leads (growth, health, calm); NHS blue anchors it (trust, credibility); a whisper of coral keeps it human. Neutrals are slate, text is off-black. These values are mirrored as CSS variables in src/index.css — that file is the source of truth in code.

Where each colour goes

Colour Use for Avoid
Emerald Primary buttons, active nav, progress bars, "counted", success states. Long body text at 500/600 weight (contrast) — step to 700.
NHS blue Text links, secondary buttons, info states, headers, the app wordmark. Competing with emerald as a second "primary" CTA on the same view.
Coral Self-care / wellbeing touches, gentle encouragement, one small highlight. Anything >10% of a screen; never for errors (that's red).
Ink Headings and primary text — the softened off-black. Pure #000; harsh on screens and clashes with the cool palette.
Slate Secondary text, borders, surfaces, disabled states. Letting it become the whole app — that's the "dull" trap.
Source of truth in code: src/index.css (@theme block). Utilities follow Tailwind naming — bg-primary-600, text-secondary-600, text-ink, bg-accent-400. Keep this file and the CSS in sync.
